# Financial Decision Log v1

Status: operating draft. Review required before public payment launch.

This log turns the financial model into decisions that can be tracked,
approved, revised, and implemented.

## Decision 001: What ABC4RD Sells

ABC4RD Academy sells:

- education;
- course cohorts;
- evidence review;
- completion certificate review;
- practical lab and demo setup;
- research bulletins;
- institutional learning infrastructure;
- language/native review support.

ABC4RD Academy does not sell:

- investments;
- Academy tokens;
- token launches;
- trading signals;
- guaranteed employment;
- accredited degrees unless separately approved;
- legal, tax, investment, medical, or financial advice.

## Decision 002: First Paid Offers

Start with three paid offers:

| Offer | Primary buyer | Price range |
| --- | --- | --- |
| Blockchain Foundations Cohort | individual learners | USD 49-149 |
| Certificate Review Batch | learners and institutions | USD 25-75 per review |
| Institutional Academy Starter Pack | universities, NGOs, companies | USD 1,500-50,000 |

Free self-study remains available as the public entry layer.

## Decision 003: Buyer Segments

Prioritize buyer segments in this order:

1. Individual learners.
2. Student groups.
3. Universities and training centers.
4. Companies.
5. NGOs, foundations, and sponsors.
6. Regional education partners.

## Decision 004: Language First, Territory Second

ABC4RD should package the Academy by language groups first:

- English as canonical source;
- Russian;
- Spanish;
- Arabic;
- French;
- Chinese.

Territory-specific selling comes later because it creates payment, tax,
consumer-protection, sanctions, advertising, and crypto legality questions.

## Decision 005: Payment Default

Fiat is the default payment path.

Crypto is allowed only as a reviewed pilot for education payments. Crypto must
not create an investment product, custody service, token sale, or token-gated
credential.

## Decision 006: Value Types

ABC4RD can receive more than two kinds of value:

| Type | Role |
| --- | --- |
| Fiat | primary operating revenue |
| Crypto assets | reviewed pilot payment path |
| Grants | restricted or unrestricted program funding |
| Sponsorship | named education support, no endorsement |
| In-kind value | cloud credits, tools, venues, equipment |
| Service exchange | scoped expert work or localization |
| Equity/revenue share | separate legal structure only |

## Decision 007: Where Money Goes

Incoming value is assigned to a ledger bucket:

| Bucket | Target |
| --- | --- |
| Operating reserve | 60-80% |
| Teacher/contractor payments | 10-25% |
| Scholarship fund | 5-15% |
| Crypto treasury experiment | 0-10% |
| Emergency reserve | 5-10% |

No payment should be treated as available for spending until the payment record,
refund status, and compliance status are recorded.

## Decision 008: Launch Gate

Before public paid launch, complete:

- one approved pricing page;
- one fiat payment rail;
- one invoice/receipt template;
- one refund rule;
- one scholarship ledger;
- one crypto pilot decision;
- one maintainer/legal/accounting review pass.
