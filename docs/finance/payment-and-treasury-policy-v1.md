# Payment And Treasury Policy v1

Status: draft for legal, tax, and accounting review.

## Default Rule

Use fiat first. Treat crypto as an optional, reviewed payment channel.

## Accepted Value Types

| Type | Status | Notes |
| --- | --- | --- |
| Fiat | default | invoices, receipts, accounting-ready |
| Crypto | pilot only | requires wallet, tax, AML/sanctions, accounting review |
| Grants | allowed after review | document restricted vs unrestricted use |
| Sponsorship | allowed after review | no editorial control or endorsement claim |
| In-kind | allowed after review | document fair value and scope |
| Service exchange | limited | written scope required |
| Equity/revenue share | separate structure | not part of core Academy payments |

## Fiat Intake

Minimum record:

- invoice ID;
- payer;
- payer country if required;
- offer purchased;
- currency;
- amount;
- date;
- payment method;
- refund status.

## Crypto Intake

Minimum record:

- invoice ID;
- payer identity where required;
- asset;
- network;
- amount;
- fiat value at receipt time;
- transaction hash;
- receiving wallet;
- compliance status;
- conversion/hold decision.

## Crypto Restrictions

- No Academy token sale.
- No investment product.
- No custody for learners.
- No token balance as certificate proof.
- No mainnet demo token launch.
- No promise of price appreciation.

## Treasury Buckets

| Bucket | Target |
| --- | --- |
| Operating reserve | 60-80% |
| Teacher/contractor payments | 10-25% |
| Scholarship fund | 5-15% |
| Crypto treasury experiment | 0-10% |
| Emergency reserve | 5-10% |

## Review Triggers

Require review before:

- accepting crypto from a new jurisdiction;
- publishing territory-specific prices;
- accepting a sponsorship with public naming;
- issuing refunds in crypto;
- retaining volatile crypto on the balance sheet;
- using any payment method that creates custody, money transmission, or
  broker-like obligations.
