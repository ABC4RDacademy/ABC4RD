# Release Checklist

Use releases to create durable public milestones. Releases make ABC4RD Academy
look like a maintained institution, not a scattered file dump.

## When To Create A Release

Create a release when a repository has one of:

- first source-backed curriculum milestone;
- reviewed source map;
- completed essay package;
- verified literature index batch;
- external contribution recorded;
- public roadmap milestone closed.

## Release Naming

Recommended pattern:

```text
v0.1-source-map
v0.2-first-lessons
v0.3-student-exercises
v0.4-review-ready
v1.0-public-track
```

## Release Notes Template

```text
## Summary

What changed and why this milestone matters.

## Added

- file or feature
- file or feature

## Verified Sources

- source
- source

## Student Tasks

- task
- task

## Risks / Boundaries

- medical, financial, safety, IP, or license boundary

## Next Milestone

What comes next.
```

## Release Gate

Before release:

- [ ] README is current.
- [ ] Source list is current.
- [ ] License/disclaimer exists.
- [ ] No private data is present.
- [ ] No copied third-party material is present.
- [ ] Open risks are documented.
- [ ] Next milestone is stated.
