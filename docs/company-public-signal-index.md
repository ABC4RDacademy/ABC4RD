# Company Public Signal Index

ABC4RD Academy needs a public ranking system that is useful, source-backed, and
defensible. The goal is not to declare which company is "best." The goal is to
publish a repeatable weekly view of who is showing the strongest public signal
across blockchain, digital infrastructure, and frontier innovation.

This index is designed to reduce avoidable conflict:

- the formula is public;
- the inputs are public;
- the categories are fixed;
- the score can be explained line by line;
- ties and disputes are handled by rule, not mood.

## What This Index Measures

The index measures public operating signal, not valuation.

It does not rank:

- token price;
- market cap alone;
- private revenue;
- rumor-driven hype;
- claimed partnerships without public proof.

It does rank:

- official product and infrastructure activity;
- public technical surface;
- public event and conference presence;
- documentation and education friendliness;
- collaboration readiness for respectful, non-spam contact.

## Scope

ABC4RD should maintain three baskets:

1. Blockchain and digital asset infrastructure
2. Open digital infrastructure and compute
3. Frontier applied innovation

Each basket can produce its own Top 20. After coverage improves, ABC4RD can
publish an overall cross-basket Top 20 using the same index family.

## Source Policy

Allowed sources:

- official company websites;
- official product documentation;
- official GitHub organizations and repositories;
- official conference pages;
- official event schedules and press pages;
- official investor-relations pages;
- official standards bodies and foundation pages.

Use third-party media only as a pointer. Final scoring should prefer official
sources.

## The Five Indices

Each entity receives five scores from `0` to `20`.

### 1. Technical Surface Index (TSI)

Measures whether the entity has a serious public technical footprint.

Signals:

- clear product or platform pages;
- technical docs or developer portal;
- public architecture, API, SDK, or protocol materials;
- evidence of serious infrastructure, not only marketing pages.

### 2. Open Activity Index (OAI)

Measures public builder-facing openness.

Signals:

- official GitHub organization or repositories;
- release notes or changelogs;
- public issue or community workflow;
- open-source tools, specs, standards, or reference implementations.

### 3. Event Presence Index (EPI)

Measures visible participation in the public ecosystem.

Signals:

- official conference appearances;
- official sponsorship pages;
- official workshops, summit sessions, or public talks;
- official event archives or recurring programs.

### 4. Knowledge Access Index (KAI)

Measures whether an outside learner, researcher, or partner can actually learn
from the entity without private access.

Signals:

- high-quality docs;
- official guides or onboarding;
- educational content, webinars, or explainers;
- public reports, papers, or implementation notes.

### 5. Collaboration Readiness Index (CRI)

Measures whether ABC4RD can approach the entity carefully and publicly.

Signals:

- official contact routes;
- official event forms;
- public partner, press, or community channels;
- clear program structure for feedback, grants, contributors, or guest contact.

## Formula

Base weekly score:

```text
Weekly Public Signal Score = TSI + OAI + EPI + KAI + CRI
```

Maximum score: `100`

## Tie-Break Rules

If two entities have the same total:

1. Higher `TSI` wins.
2. If still tied, higher `OAI` wins.
3. If still tied, higher `EPI` wins.
4. If still tied, the entity with the newer official signal date wins.
5. If still tied, alphabetical order.

## Weekly Publication Rule

Every weekly ranking must include:

- the ranking date;
- the basket being ranked;
- the exact formula;
- the tracked source file or table;
- a note that inclusion is not endorsement or partnership.

## Public Dispute Rule

If an entity asks why it is ranked at a given place, ABC4RD should respond with:

1. The five component scores.
2. The public links used for scoring.
3. The date range of the review.
4. The next public signal that could improve the score.

ABC4RD should never argue emotionally about rank position. The system should
answer with transparent components and source links.

## Outreach Rule

The ranking system does not authorize mass outreach.

Use this order:

1. Watch
2. Verify
3. Cite
4. Contribute
5. Ask for accuracy feedback
6. Invite only after public value exists

Allowed early outreach:

- accuracy correction requests;
- preferred-link requests;
- docs contribution notes;
- guest-session invitations after useful public material exists.

Not allowed:

- mass email;
- fake partnership language;
- automated cold messaging at scale;
- public claims based on private replies.

## Weekly Workflow

Monday:

- refresh official sources;
- update the watchlist table;
- add or remove tracked entities;
- rescore the five indices.

Wednesday:

- review conferences, summits, webinars, and official event pages;
- identify new public learning or contribution opportunities;
- draft outreach notes only when warranted.

Friday:

- publish the weekly Top 20;
- publish the "why this changed" paragraph;
- publish a short watchlist delta note.

## Expansion Plan

Phase 1:

- publish a seed Top 20 for blockchain and digital asset infrastructure;
- keep a cross-sector watchlist for future expansion.

Phase 2:

- publish separate Top 20 lists for open compute and applied innovation;
- normalize cross-track comparisons.

Phase 3:

- generate weekly tables automatically from a maintained CSV or JSON source;
- publish a short public digest alongside the ranking.

## Standing Disclaimer

Inclusion in the index means "relevant public ecosystem target for research,
verification, contribution, or respectful contact."

It does not mean:

- partner;
- sponsor;
- customer;
- endorsement;
- formal relationship.
