# Blockchain Career Map

ABC4RD Academy helps students understand blockchain careers through skills,
public repositories, first tasks, and responsible boundaries.

## Career Paths

| Path | Learn | Read / watch | First task |
|---|---|---|---|
| Bitcoin developer | C++, Rust, protocol basics, UTXO, transactions, testing | Bitcoin Core, Bitcoin Optech, LDK, BTCPay Server | Fix docs, summarize BIP, test wallet flow |
| Smart contract developer | Solidity, EVM, Foundry/Hardhat, testing, security basics | Ethereum docs, OpenZeppelin, Foundry, Hardhat | Write toy contract test and risk note |
| Blockchain security analyst | Threat modeling, smart contract vulnerabilities, custody risks | Trail of Bits blogs, OpenZeppelin, security reports | Review a simple contract checklist |
| Protocol researcher | Cryptography, consensus, distributed systems, incentives | Bitcoin research, Ethereum research, academic papers | Summarize one paper with assumptions |
| RWA analyst | Tokenization, custody, legal claims, oracles, attestations | Chainlink, Centrifuge, Maker/Sky, Ondo, Polymesh | Compare token vs legal ownership claim |
| DAO operator | Governance, treasury, multisig, Snapshot, delegation | Aragon, Snapshot, Safe, Tally | Draft a proposal lifecycle map |
| Web3 product manager | Wallets, UX, custody, data ownership, community | Wallet docs, dapp docs, DAO forums | Write user-risk map |
| Tokenization compliance analyst | Legal boundaries, disclosures, jurisdiction, reporting | Public regulatory and standards sources | Mark claims requiring legal review |
| Open-source contributor | GitHub, docs, issues, PR etiquette | Project contribution guides | Open a docs PR or source issue |
| Blockchain educator | Curriculum design, source review, examples, ethics | ABC4RD curricula, public courses | Improve glossary or reading list |
| AI + blockchain infrastructure engineer | AI agents, verifiable compute, provenance, signing, logs | MLCommons, CNCF, OpenInfra, model provenance sources | Design a compute record template |

## Shared Skills

- Git and GitHub.
- Markdown documentation.
- Basic cryptography vocabulary.
- Threat modeling.
- Source verification.
- Reading standards.
- Writing narrow claims.
- Respectful open-source communication.

## Student Ladder

1. Read and summarize.
2. Verify source.
3. Add glossary term.
4. Improve documentation.
5. Create exercise.
6. Open internal issue.
7. Open external docs issue.
8. Submit external PR.
9. Review another student.
10. Maintain a learning track.

## Portfolio Artifacts

Students should build:

- glossary contributions;
- source-review issues;
- reading notes;
- toy examples;
- risk reviews;
- external PRs;
- digest summaries;
- public learning maps.
