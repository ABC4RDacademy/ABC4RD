# GitHub Roadmap

This roadmap prepares ABC4RDacademy for public development without leaking private materials or overstating unverified claims.

## Phase 1: Immediate cleanup

- Publish platform-safe repository structures.
- Remove unsafe filenames from active repositories.
- Keep legacy materials in archives or documented history, not as fragile root files.
- Add `.gitignore`, issue templates, contribution docs, and safety policy.
- Check all public text for source discipline and international positioning.

## Phase 2: Public profile

- Publish `ABC4RDacademy/.github` with a clear organization profile.
- Pin the main repositories: `ABC4RD`, `LLMAIX2001a`, `abc4rd-research`, `abc4rd-community`.
- Add concise repository descriptions and topics.
- Enable Issues and Discussions.
- Create the six discussion categories listed in `docs/discussions.md`.

## Phase 3: Content pipeline

- Keep canonical long-form content in Markdown.
- Track status as `draft`, `review`, `approved`, `published`, or `archived`.
- Use GitHub Issues for article proposals and source submissions.
- Generate Telegram, Discord, X, and website drafts only after source review.
- Require human approval before any public post.

## Phase 4: Site integration

- Connect `ABC4RD` to the website repository or static site stack once access is confirmed.
- Add pages for About, Courses, Research, Evidence, Community, and Contact.
- Use structured content files where possible.
- Add OpenGraph images and link previews.
- Run link and Markdown checks in GitHub Actions.

## Phase 5: Telegram and Discord integration

- Store bot tokens and webhooks only in GitHub Secrets or local `.env`.
- Generate channel drafts from approved Markdown.
- Keep posting manual until the review process is stable.
- Record published URLs back into the relevant issue or content file.

## Phase 6: Conference activation

- Track Bitcoin 2026, Bitcoin Asia, Bitcoin Amsterdam, Bitcoin MENA, and adjacent events with public sources.
- Open one event issue per opportunity.
- Connect every event target to a course, workshop, article, or learning module.
- Avoid claiming partnership unless confirmed publicly or by approved written agreement.

## Phase 7: Open-source course development

- Develop LLMAIX2001a as the first course track.
- Split lessons into modules and labs.
- Add beginner-friendly issues.
- Use public pull requests for course content, labs, translations, and evidence updates.
- Publish milestone notes quarterly.
