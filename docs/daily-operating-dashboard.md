# ABC4RD Daily Operating Dashboard

Status: `source`

![ABC4RD daily operating dashboard preview](../assets/article-previews/daily-operating-dashboard.svg)

This dashboard is the public daily control surface for ABC4RD Academy.

It turns the 24/7 operating rhythm into a visible daily package that can be
repeated without noise.

## Purpose

ABC4RD should not wait for Friday to become visible.

The daily dashboard exists to keep:

- source verification moving;
- curriculum updates steady;
- issue and draft work organized;
- publication readiness visible;
- weekly ranking work prepared before the weekly release window.

## Daily Package Structure

Each day should try to produce five small outputs:

1. one source block
2. one curriculum block
3. one contribution block
4. one external documentation or ecosystem block
5. one digest block

## Daily Dashboard

| Block | Minimum output | Public rule |
| --- | --- | --- |
| Source block | Verify or annotate 1-3 public sources | Official-source links only |
| Curriculum block | Improve one module, lesson, glossary, or exercise | Must point to a real file |
| Contribution block | Open, refine, or close one work item | Must map to an issue, PR, or draft |
| External block | Identify one external contribution or event signal | Draft-only unless approved |
| Digest block | Record what changed and what needs review next | Must stay factual and concise |

## Ranking Preparation Rule

On non-Friday days, ranking work should focus on preparation:

- source-table cleanup;
- watchlist additions;
- localization gaps;
- preview coverage;
- delta baseline health;
- automation readiness;
- draft notes for the next ranking cycle.

On Friday, that preparation converts into official publication.

## Daily Outputs That Count

- a real source row added or corrected
- a curriculum paragraph improved
- a preview image created or normalized
- a localization draft added
- a ranking CSV repaired
- a delta draft prepared
- a public dashboard link improved

## Daily Outputs That Do Not Count

- empty commits
- duplicate notes with no new source
- vague partner claims
- mass outreach
- unsupported ranking movement claims

## Safety Note

The daily dashboard is an execution discipline layer.

It does not authorize:

- outbound mass messaging;
- unsupported public claims;
- publication without review where review is required.
