# Repository Scorecard

ABC4RD Academy uses this scorecard to decide whether a repository is ready to
be featured, shared with experts, linked from the main hub, or used for student
onboarding.

## Scoring

| Score | Meaning |
|---|---|
| 0 | Missing or unsafe |
| 1 | Draft exists |
| 2 | Usable but thin |
| 3 | Public-ready |
| 4 | Strong |
| 5 | Reference-quality |

## Criteria

| Area | Question | Score |
|---|---|---|
| Purpose | Does the README explain the repository in the first paragraph? | 0-5 |
| Audience | Are students, contributors, experts, or partners clearly identified? | 0-5 |
| Sources | Are primary public sources listed with verification status? | 0-5 |
| Education | Are modules, exercises, glossary, and expected outputs present? | 0-5 |
| Trust | Are license, disclaimer, source policy, and safety boundaries present? | 0-5 |
| Contribution | Are issues, contribution guide, and PR expectations clear? | 0-5 |
| Reputation | Are recent commits, closed issues, digests, or releases visible? | 0-5 |
| Externality | Does the repository connect to external open-source projects responsibly? | 0-5 |
| IP safety | Does it avoid leaking patentable, trade-secret, or third-party restricted material? | 0-5 |
| Maintenance | Is there a clear next milestone and owner role? | 0-5 |

## Interpretation

| Total | Status | Action |
|---|---|---|
| 0-15 | Hold | Do not feature. Fix safety, purpose, and source gaps first. |
| 16-29 | Draft | Keep public if safe, but do not promote heavily. |
| 30-39 | Active | Suitable for student tasks and internal roadmap links. |
| 40-46 | Strong | Suitable for expert feedback and ecosystem references. |
| 47-50 | Reference-quality | Suitable for pinned profile, releases, and public showcases. |
