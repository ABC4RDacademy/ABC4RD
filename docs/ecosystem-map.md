# ABC4RD Academy Ecosystem Map

ABC4RD Academy maps the global technical ecosystem so students, researchers,
contributors, and partners can understand where open-source education connects
to Bitcoin, blockchain, AI, open compute, digital health, robotics, digital
manufacturing, and nanomaterials.

This map is not a partnership list. Inclusion means "relevant public ecosystem
target for research, citation, contribution, or respectful feedback."

## Integration Principles

- Public value before outreach.
- Source-backed references before claims.
- No endorsement language without explicit approval.
- No mass tagging, mass email, or automated outreach.
- Prefer issues, docs fixes, source suggestions, and narrow accuracy feedback.
- Move actionable feedback into Bitrix for internal follow-up.

## Public Visibility Loop

1. Add a public source to an index.
2. Open a source-review issue.
3. Improve one ABC4RD module or glossary entry.
4. Find one upstream docs contribution opportunity.
5. Publish a weekly digest.
6. Route actionable feedback to Bitrix.
7. Ask for narrow accuracy feedback only after the public artifact is useful.

## Follow / Watch Strategy

ABC4RD Academy should follow and watch projects to learn, verify, and
contribute. It should not follow accounts just to look bigger.

Useful signals:

- new beginner issues;
- release notes;
- conference materials;
- documentation gaps;
- standards updates;
- research calls;
- hackathons and workshops;
- source changes affecting ABC4RD curricula.

## Data Statuses

| Status | Meaning |
|---|---|
| `verified` | Official public source confirms the entry |
| `requires verification` | Needs source, GitHub link, or event-page review |
| `watch` | Useful to monitor but not ready for outreach |
| `contribution target` | Suitable for small docs/source PR |
| `do not contact yet` | Needs stronger ABC4RD public artifact first |
