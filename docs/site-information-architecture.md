# Site Information Architecture

This document defines the minimum public website structure for ABC4RD Academy.
The first website should help visitors understand the initiative, find the
flagship course, inspect evidence, and join public work without private
outreach.

## Pages

| Page | Purpose | Primary action |
| --- | --- | --- |
| About | Explain ABC4RD history, relaunch, and source discipline | Read the public positioning |
| Courses | Show active and planned learning tracks | Open `LLMAIX2001a` |
| LLMAIX2001a | Present the flagship course and runnable labs | Start Module 01 |
| Frontier AI | Explain agents, AGI literacy, ASI safety, and blockchain trust | Read the innovation strategy |
| Community | Route learners, contributors, researchers, educators, and partners | Join GitHub Discussions |
| Research / Evidence | Keep claims source-backed and reusable | Open the research repository |
| Contact | Point to approved public channels | Use GitHub Issues or community channels |

## First-screen message

ABC4RD Academy is an international blockchain and AI education initiative with a
public 2017 origin, a flagship AI course, and a source-backed approach to
frontier AI, Bitcoin education, and blockchain trust.

## Content rules

- Put the flagship course in the first viewport or first content section.
- Link claims to public sources or repository evidence.
- Do not claim partnership, sponsorship, endorsement, AGI, or ASI capability.
- Keep external outreach draft-first and approval-gated.
- Link to GitHub Issues before asking visitors to join private channels.

## Required links

- `ABC4RD`: main hub and operating loop.
- `LLMAIX2001a`: flagship course and Module 01 lab.
- `abc4rd-research`: source map, citation pack, media kit.
- `abc4rd-community`: onboarding, roles, community operations.
- `docs/frontier-innovation-strategy.md`: agents, AGI, ASI, blockchain trust.
