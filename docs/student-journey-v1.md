# ABC4RD Student Journey v1

Status: v0.1 draft.

Time to prepare MVP: 1-3 days.

Estimated operating cost: $0 for GitHub-based workflow.

## Purpose

The student journey connects ABC4RD Academy infrastructure into a real learning path.

The first journey is:

`read -> build -> submit -> review -> certify -> portfolio`.

## Step 1: Choose a Track

First track:

- Blockchain Foundations: From Bitcoin to Verifiable Infrastructure.

Student chooses:

- beginner path;
- developer path;
- research path;
- product/venture path.

## Step 2: Read and Watch

Student reads:

- one course module;
- three public sources;
- one ABC4RD glossary note;
- one relevant lab page.

Optional:

- watch a lecture or media note when available.

## Step 3: Complete Assignment

Student creates one public artifact:

- summary;
- glossary entry;
- diagram;
- use-case evaluation;
- security checklist;
- short code explanation;
- source review.

## Step 4: Submit Through GitHub

Student opens a pull request with:

- assignment file;
- sources;
- disclaimer;
- one open question.

Reviewers check:

- source quality;
- clarity;
- originality;
- no plagiarism;
- no investment claims;
- no unsupported partnership claims;
- no private data.

## Step 5: Receive Review

Review types:

- peer review;
- mentor review;
- maintainer review;
- domain review when needed.

Review result:

- accepted;
- needs revision;
- requires source verification;
- out of scope;
- rejected for safety/legal reasons.

## Step 6: Certificate

After completing required work, the student receives:

- certificate ID;
- completion page;
- QR/link verification;
- course name;
- completion date;
- review status;
- disclaimer.

First certificate type:

- ABC4RD Blockchain Foundations Completion Certificate.

## Step 7: Portfolio

Student adds to portfolio:

- course completed;
- GitHub PR link;
- assignment link;
- certificate verification link;
- one reflection note;
- next learning goal.

## Step 8: Next Path

Possible next steps:

- Solidity testnet challenge;
- Bitcoin open-source reading path;
- DAO governance challenge;
- RWA use-case evaluation;
- cybersecurity baseline;
- AI website builder project;
- mentor review session.

## Minimal Operating Loop

Every week:

1. publish challenge;
2. accept submissions;
3. review PRs;
4. issue certificates for accepted work;
5. update student portfolio templates;
6. publish weekly digest.
