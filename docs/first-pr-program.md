# First PR Program

The ABC4RD Academy First PR Program helps students move from reading to public
open-source contribution.

## Goals

- Teach GitHub basics.
- Build confidence.
- Improve ABC4RD Academy materials.
- Make respectful upstream documentation contributions.
- Create public proof of learning.

## Internal Beginner Issues

Create or maintain issues for:

1. Add one glossary term.
2. Verify one source.
3. Summarize one public article.
4. Fix one broken link.
5. Improve one README section.
6. Add one exercise.
7. Rewrite one hype claim into a narrow claim.
8. Add one conference source.
9. Add one university source.
10. Review one discussion and summarize next steps.

## External Beginner Targets

Candidate project types:

- documentation repositories;
- glossary projects;
- beginner tutorials;
- broken-link fixes;
- translation reviews where allowed;
- source clarification issues.

Candidate ecosystems:

- Bitcoin / Lightning / BTCPay.
- Ethereum tooling.
- Hyperledger.
- CNCF / Kubernetes glossary.
- ROS 2 / Gazebo.
- OpenMRS / FHIR.
- Materials Project / NOMAD.

## PR Template

```text
This PR makes a small documentation improvement in [file/section].

Context:
While reviewing beginner-facing material for source-backed education, I noticed
that [specific point] could be clearer for new readers.

Changes:
- [change 1]
- [change 2]

No behavior changes. Documentation only.

Thank you for maintaining this project.
```

## Weekly Review

Each week:

- review open student PRs;
- close completed internal issues;
- identify one external docs target;
- publish a short digest;
- update the contribution ladder.
