# GitHub Watchlist

ABC4RD Academy uses GitHub watch/star/follow activity to find source updates,
documentation gaps, beginner issues, and open-source contribution routes.

This is not a popularity campaign.

## Watch Categories

| Category | Examples | Why watch |
|---|---|---|
| Bitcoin protocol | bitcoin/bitcoin, bitcoin-core/gui, bitcoin-core/secp256k1 | Protocol literacy and docs references |
| Bitcoin education | bitcoinops/bitcoinops.github.io, chaincodelabs, brink | Curriculum sources and reading paths |
| Lightning | lightningdevkit/rust-lightning, lightningnetwork/lnd, ElementsProject/lightning | Lightning curriculum and contribution routes |
| Payments / self-custody | btcpayserver/btcpayserver, btcpayserver/btcpayserver-doc | Beginner docs and wallet/payment literacy |
| Ethereum tooling | ethereum, foundry-rs/foundry, NomicFoundation/hardhat | Smart contract education and tooling comparisons |
| Hyperledger / LF Decentralized Trust | hyperledger repositories | Enterprise blockchain and identity education |
| RWA tooling | chainlink, centrifuge, polymesh, tokenization-related public repos | RWA literacy and source mapping |
| DAO tooling | aragon, snapshot-labs, safe-global, tallyxyz | Governance tooling and treasury education |
| Digital health | openmrs/openmrs-core, hapifhir/hapi-fhir, GNUHealth | FHIR/open health systems and docs contributions |
| Robotics | ros2/ros2, ros2/ros2_documentation, gazebosim, moveit | Simulation-first robotics curriculum |
| Open compute | kubernetes/kubernetes, cncf/glossary, openstack, jupyter | Reproducible compute and infrastructure literacy |
| Materials data | materialsproject, FAIRmat-NFDI, Materials-Consortia | Nanomaterials and simulation literacy |

## Watch Signals

When watching a repository, record:

- official repo link;
- contribution guide;
- issue labels for beginners;
- docs folder;
- discussion forum;
- recent release activity;
- whether ABC4RD has a relevant curriculum module;
- possible small contribution.

## Follow Rules

- Follow only relevant maintainers, organizations, and projects.
- Do not follow/unfollow in bulk.
- Do not tag people without context.
- Do not represent stars/follows as partnerships.
- Move useful opportunities into GitHub Issues or private ABC4RD CRM tasks.
