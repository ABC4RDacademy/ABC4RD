# Blockchain Foundations Submission Review Checklist

Status: v0.1 draft.

Use this checklist to review student submissions for:

`Blockchain Foundations: From Bitcoin to Verifiable Infrastructure`.

## Basic Acceptance

- [ ] Submission uses the template.
- [ ] Student name or public handle is present.
- [ ] GitHub username is present.
- [ ] Selected module is marked.
- [ ] Summary is 250-500 words.
- [ ] At least 3 public sources are cited.
- [ ] One artifact is included: table, diagram, glossary entry, checklist, DAO proposal, use-case evaluation, or code explanation.
- [ ] One open question is included.
- [ ] Portfolio note is included.
- [ ] Disclaimer is included.

## Source Quality

- [ ] Sources are public.
- [ ] Sources are relevant to the selected module.
- [ ] Sources are not pirated PDFs, copied books, private files, or unclear reposts.
- [ ] Claims are not presented without source support.

## Safety and Policy

- [ ] No private keys, seed phrases, passwords, API tokens, or secrets.
- [ ] No private student data.
- [ ] No private contact data.
- [ ] No financial or investment advice.
- [ ] No legal, medical, tax, or security audit advice.
- [ ] No production deployment approval.
- [ ] No fake partnership, endorsement, or affiliation claim.

## Technical Review

- [ ] Blockchain capability is not overstated.
- [ ] Token and legal ownership are separated where relevant.
- [ ] Off-chain/oracle risk is identified where relevant.
- [ ] Localnet/testnet/mainnet boundaries are clear where relevant.
- [ ] Security limitations are mentioned where relevant.

## Review Decision

Choose one:

- `accepted`;
- `minor revision requested`;
- `major revision requested`;
- `requires source verification`;
- `rejected: unsafe claims`;
- `rejected: private data`;
- `rejected: plagiarism or unclear attribution`.

## Reviewer Comment Template

```text
Thank you for the submission.

Review result: [accepted / revision requested / requires verification].

What works:

What needs improvement:
- ...

Required next step:
- ...
```
