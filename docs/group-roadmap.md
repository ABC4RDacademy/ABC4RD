# ABC4RD Academy Group Roadmap

This roadmap turns the GitLab group into a living education operating system.

## 30 days

- Keep `ABC4RD` as the public command center.
- Publish the GitLab Pages navigation site.
- Attach labels and `Activation Sprint 1` to the first work items.
- Add CI lint to the main, course, research, community, and Bitcoin repositories.
- Open the first 30 starter issues.
- Publish Weekly Recap 1 and a monthly operating note.
- Start the LLMAIX2001a module map and first four labs.

## 90 days

- Complete the LLMAIX2001a 17-module map.
- Publish beginner, developer, researcher, and teacher tracks.
- Create 50 beginner issues, 20 research verification issues, 20 translation issues, and 10 lab/demo issues.
- Build the evidence policy into every public claim path.
- Create a public metrics dashboard updated every week.
- Prepare partner drafts for GitLab Education, universities, Bitcoin education, open-source AI, and science communities.

## 180 days

- Ship the first complete course cohort pathway.
- Publish multilingual source-parity translations for the core positioning text.
- Run at least one public event or workshop linked to course artifacts.
- Maintain an active issue board with draft, evidence, review, approved, and published lanes.
- Track contributors, merge requests, course modules, citations, translations, stars, forks, and published artifacts.

## Operating rule

Useful public work beats promotion. Every visible update should help a learner, researcher, developer, community builder, or partner understand what to do next.
