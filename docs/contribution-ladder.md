# Contribution Ladder

ABC4RD Academy uses a visible ladder so students can move from reading to real
open-source contribution without creating noise for maintainers.

## Levels

| Level | Name | Output | Review standard |
| --- | --- | --- | --- |
| 1 | Read and summarize | 150-300 word source note | Accurate, cited, no copying |
| 2 | Verify source | Source review issue or table entry | URL, author/institution, status, date checked |
| 3 | Add glossary term | One term with source | Clear beginner language |
| 4 | Improve documentation | Small internal PR | Focused change, no broad rewrite |
| 5 | Create exercise | Student task with expected output | Practical, source-backed, safe |
| 6 | Open external issue | One concrete upstream issue | Specific problem, respectful tone |
| 7 | Submit external PR | Small upstream documentation PR | Maintainer rules followed |
| 8 | Review another student | Constructive PR or issue review | Specific, kind, evidence-backed |
| 9 | Lead a research track | Track update, digest, or module | Repeatable process and source discipline |

## Rules

- Start with internal tasks before external outreach.
- Never open vague upstream issues.
- Prefer documentation, source clarification, and beginner notes as first
  external contributions.
- Do not ask maintainers to promote ABC4RD Academy.
- Record every external contribution in the weekly digest.

## Student First Week

1. Pick one track from `docs/research-tracks.md`.
2. Read two verified sources.
3. Add one source note.
4. Add one glossary term.
5. Open one internal issue with a specific improvement.
6. Prepare, but do not send, one external contribution draft.
