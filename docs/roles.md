# Public Roles

ABC4RD Academy can describe roles before assigning named people. The goal is to
make work reviewable and reduce confusion.

| Role | Responsibility | Public output |
| --- | --- | --- |
| Research maintainer | Source maps, literature index, evidence policy | Reviewed sources, research issues, digests |
| Curriculum maintainer | Learning modules, exercises, glossary quality | Curriculum PRs and module milestones |
| Source verification reviewer | Source status, license checks, citation quality | `verified`, `requires verification`, `rejected`, `license unclear` decisions |
| Outreach coordinator | Feedback requests, expert messages, event notes | Draft-only outreach, no mass messages |
| Student onboarding coordinator | First issues, contribution ladder, beginner paths | Student tasks and onboarding docs |
| Technical reviewer | Engineering accuracy, safety, standards, automation | PR review, CI checks, external contribution review |

## Operating Rule

One person may hold multiple roles, but every public artifact should still show
which role is responsible for the next review.
