# ABC4RD Operating Dashboard

This dashboard is the public control surface for keeping ABC4RD visible,
credible, and active across GitHub, GitLab, Bitbucket, website, Telegram,
Discord, and X.

## Public signal

| Surface | Role | Current operating rule |
| --- | --- | --- |
| Website | Canonical public narrative | Publish only approved, source-backed pages |
| GitHub | Public source of truth and community work | Issues, discussions, labels, pull requests |
| GitLab | Sprint board, CI mirror, and Pages dashboard | Protected branch, validation pipeline, mirror checks |
| Bitbucket | Atlassian-facing public mirror and review surface | Public repository, branch restrictions, PR template, pipeline |
| Telegram | Short public updates | Draft-first, approved links only |
| Discord | Community discussion | Draft-first, moderated, no private claims |
| X | Discovery and recap channel | Short threads with source-backed claims |

## Weekly cadence

| Weekly output | Artifact | Approval |
| --- | --- | --- |
| Course update | LLMAIX2001a module issue, PR, or learning note | Maintainer review |
| Research source | Citation block, evidence note, or source review | Source review |
| Event watch | Event issue with official URL and education angle | Human approval before outreach |
| Partner draft | Target, reason, message, artifact, follow-up owner | Human approval before sending |
| Community recap | Telegram, Discord, X, GitHub/GitLab/Bitbucket draft | Human approval before posting |

## Number-one standard

- Clear public positioning in the first screen of every repository.
- No unverified partnership claims.
- Public work trail every week.
- One useful learning artifact before every outreach message.
- Repository checks green before public promotion.
- Tokens, webhooks, private contracts, emails, and CRM notes stay out of git.

## Current task routing

Bitbucket issue creation is blocked for the current repository token, so task
tracking should use GitHub Issues or GitLab issues until Bitbucket issue access
is explicitly enabled. Bitbucket should still hold the mirror, README, PR
template, branch restrictions, pipeline, and the starter issue drafts in
`docs/bitbucket-starter-issues.md`.

## Partner approval gate

Before any external message, record:

- target person, organization, community, or event;
- public source URL;
- reason ABC4RD should engage;
- learning artifact to share;
- draft message;
- approval owner;
- follow-up date.

No mass invites, mass subscriptions, event applications, or partner outreach
without this gate.
