# ABC4RDacademy Organization Structure

## Repositories

### `.github`

Organization profile repository. Contains `profile/README.md` and shared GitHub defaults.

### `ABC4RD`

Main public hub for the academy, website direction, roadmap, publishing policy, and cross-repository coordination.

### `LLMAIX2001a`

First public AI course track. It should hold curriculum, labs, module issues, course roadmap, and course-specific partner notes.

### `abc4rd-research`

Evidence and public research repository. It should hold citation maps, source review, public media references, and media kit materials.

### `abc4rd-community`

Community and governance repository. It should hold the public charter, governance model, community operations, event playbooks, and contribution pathways.

## Platform mirrors

GitHub remains the current public source of truth. GitLab and Bitbucket may be
used as controlled mirrors, sprint workspaces, partner review spaces, or CI/CD
experiments after credentials and permissions are approved.

- GitHub: organization profile, public issues, discussions, pull requests.
- GitLab: issue board, protected branch, validation pipeline, starter issues.
- Bitbucket: repository page, branch restrictions, PR template, Jira or
  external tracker links, and Atlassian-facing review flow.

## Operating model

- Keep content work public when possible.
- Keep secrets and private outreach out of repositories.
- Use issues for concrete tasks.
- Use discussions for broad community work.
- Use pull requests for reviewable changes.
- Keep public claims tied to sources.
