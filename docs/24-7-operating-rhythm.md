# 24/7 Operating Rhythm

ABC4RD Academy should show constant movement without creating spam. The goal is
steady public proof of work, not noise.

## Daily Cycle

| Time block | Output |
|---|---|
| Source block | Verify or annotate 1-3 public sources |
| Curriculum block | Improve one module or exercise |
| Contribution block | Open, refine, or close one internal issue |
| External block | Identify one external docs contribution opportunity |
| Digest block | Record what changed and what needs review |

## Weekly Cycle

- Publish one research digest.
- Expand one research track.
- Improve two curriculum modules.
- Add five source entries.
- Close completed issues.
- Review external PR/issue status.
- Update roadmap if priorities changed.

## Monthly Cycle

- Create a release for one mature track.
- Review legal and IP boundaries.
- Archive or de-emphasize noisy repositories.
- Publish one long-form essay or whitepaper.
- Request narrow expert feedback on one source-backed module.

## Activity Rules

- No mass outreach.
- No empty commits.
- No unsupported partnership claims.
- No copying third-party materials.
- Prefer small useful changes over theatrical launches.

## Useful Public Signals

- Commits with meaningful names.
- Closed issues tied to real files.
- Source verification history.
- External documentation PRs.
- GitHub Discussions with concrete questions.
- Digests that summarize progress.
- Releases for milestones.

## Do Not Optimize For

- Stars without substance.
- Follower count without contribution.
- Tagging famous people.
- Creating more repositories than can be maintained.
- Posting faster than claims can be verified.
