# Module 08: First Open-Source Contribution

Course: Blockchain Foundations: From Bitcoin to Verifiable Infrastructure.

Status: v0.1 draft.

Time: 60-180 minutes.

## Goal

Learn how to make a small public contribution through GitHub: source note, glossary entry, typo fix, documentation improvement, issue, or pull request.

## Key Ideas

- repository;
- issue;
- branch;
- pull request;
- review;
- maintainer;
- contribution guide;
- license;
- attribution;
- code of conduct.

## Read

- ABC4RD Contribution Ladder;
- ABC4RD First PR Program;
- GitHub documentation on pull requests;
- contribution guide of one open-source project.

## Learn

Open source contribution is not only code. Good source notes, documentation fixes, glossary improvements, examples, diagrams, and beginner explanations are useful when they are accurate and respectful.

## Assignment

Choose one:

1. add a glossary term to ABC4RD;
2. add a source note;
3. improve a module explanation;
4. open a beginner-friendly issue;
5. write a short documentation PR;
6. create a student portfolio entry.

## Submission

Open a pull request with:

- title;
- short explanation;
- sources;
- disclaimer;
- one question for reviewers.

## Review Criteria

- contribution is small and focused;
- sources are cited;
- no spam or mass tagging;
- no fake affiliation or endorsement;
- tone is respectful.

## Safety Note

Do not tag external maintainers aggressively. First create useful work, then ask for accuracy feedback.
