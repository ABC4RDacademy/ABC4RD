# ABC4RD Academy Governance

ABC4RD Academy is operated as an independent research and education initiative.
This governance document explains how public work should be proposed, reviewed,
credited, and published.

## Scope

This governance applies to:

- curriculum repositories;
- research notes and source indexes;
- GitHub Pages content;
- public issues and discussions;
- outreach drafts;
- releases and milestone notes.

## Maintainer Roles

| Role | Responsibility |
|---|---|
| Research maintainer | Track architecture, source quality, research questions |
| Curriculum maintainer | Modules, exercises, glossary, student learning paths |
| Source verification reviewer | Source status, license checks, citation quality |
| Technical reviewer | Repository structure, automation, docs checks |
| Outreach coordinator | Delicate feedback requests and non-spam collaboration |
| Student onboarding coordinator | First issues, first PRs, contribution ladder |

Named maintainers may be added later. Until then, repository owners act as
interim maintainers.

## Decision Process

Routine documentation and curriculum edits can be merged by a maintainer after
review. Sensitive changes require a slower review:

- medical, clinical, or health-data claims;
- financial, investment, or custody guidance;
- safety-critical robotics or manufacturing claims;
- patent-sensitive implementation details;
- named partnerships, sponsors, or affiliations;
- use of third-party materials with unclear license status.

When in doubt, mark the issue `needs review` or `requires verification`.

## Contribution Standards

Contributions should:

- cite public sources;
- avoid unsupported hype;
- separate facts from plans;
- avoid private data and credentials;
- respect third-party licenses;
- use beginner-safe explanations where the audience is students;
- include disclaimers where finance, medicine, safety, or IP are involved.

## Source Statuses

| Status | Meaning |
|---|---|
| `verified` | Public source supports the claim and reuse boundary is clear |
| `requires verification` | Link exists but claim, license, or relevance needs review |
| `license unclear` | Do not reuse beyond link/short annotation |
| `rejected` | Source is unsuitable, private, pirated, misleading, or unsafe |

## Releases

Public releases should include:

- short summary;
- linked files changed;
- source-verification status;
- known limitations;
- next milestone;
- disclaimer when needed.

## Independence

ABC4RD Academy is independent. Do not describe external organizations,
conferences, speakers, sponsors, or projects as partners unless explicit public
approval exists.
